======================================================================================= 로컬 (현재프로젝트, commit된 대상이저장되는스테이징영역, commit한 소스보관) ======================================================================================= 1. git 사용자등록 - git config --global user.name "사용자명" - git config --global user.email "이메일주소" 2. git init : git 저장소 생성 3. git add [파일명] : 파일을 git가 추적하도록 설정 4. git commit -m "커밋메시지" : commit하기, -am : add를 함께하기 5. git status : 커밋되지 않은 변경사항을 조회하기 6. git diff : 스테이징영역과 현재 작업트리의 차이점을 보여줌 7. git checkout -- 파일명 : 스테이징이나 커밋하지않은 파일의 변경내용을 취소하고 이전 커밋상태로 돌림 8. git branch : 현재브랜치 조회 9. git branch [브랜치명] : 새브랜치 생성 10. git branch -d [브랜치명] : 브랜치명삭제 11. git checkout 브랜치명/태그명 : 해당브랜치나 태그로 작업트리변경 12. git merge 브랜치명 : 현재브랜치에 새로운브랜치를 합친다. 13. git log : 커밋로그 - git log --pretty-oneline : 한줄로 간단히보기 ======================================================================================= 원격저장소 ======================================================================================= 1. 원격저장소 접속 - git remote add [alias] [url] - git remote add origin https://github.com/80gaeng/study_hard.git 2. 원격저장소 삭제 - git remote remove [alias] - git remote remove origin 3. 원격저장소에서 내려받기 - git clone [url] [폴더명] - git clone https://github.com/80gaeng/study_hard.git 4. 원격저장소의 변경사항을 가져와서 원격브랜치를 갱신 - git fetch 5. git fetch에서하는 원격저장소의 변경사항을 가져와서 지역브랜치에 합치는작업을 한꺼번에함 - git pull 6.원격저장소로 소스올리기 - git push -u [alias] [branch name] - git push -u origin young_branch
